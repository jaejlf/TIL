# [운영체제] 프로세스&스레드, 멀티 프로세싱&멀티 스레딩

## 용어 정리
- 프로그램 : 명령어의 집합 (소스 코드와 실행 파일)
- 프로세스 : 실행 중인 프로그램 (각각의 프로세스는 독립된 메모리 공간을 할당받음.)
- 스레드 : 프로세스 내에서 실행되는 기본 단위 (프로세스와 자원 공유)
- CPU : 중앙 처리 장치, 명령어를 실행하는 역할
- 메인 메모리 : 프로세스가 CPU에서 실행되기 위해 대기하는 곳
- IO : 파일을 읽고 쓰거나, 입출력 장치와 데이터를 주고받는 것

<br>

- `한 번에 하나`의 프로그램만 실행할 수 있다.
- ⛔ IO 작업을 하는 동안 CPU는 사용되지 않기 때문에, CPU 사용률이 좋지 않다.

## 멀티 프로그래밍

![image](https://github.com/user-attachments/assets/028f1181-9b2a-4fcd-b4fe-8cc2b25dbbe6)

- 여러 개의 프로그램을 메모리에 올려놓고, `한 프로세스가 IO 작업을 시작하면 그동안 다른 프로세스를 CPU에서 실행`한다.
- ⛔특정 프로세스의 CPU 사용 시간이 길어지면, 다른 프로세스는 계속 대기하게 된다.

<br>

## 멀티 태스킹

![image](https://github.com/user-attachments/assets/b9a03617-8790-45f1-af57-2842609fed12)

- (멀티 프로그래밍과 같으나) CPU 사용 시간을 짧은 시간 간격(=quantum)으로 `쪼개서, 번갈아가면서 실행`한다. (실제로 동시에 실행되는 것은 아니지만, 사용자가 느끼기에는 동시에 실행되는 것처럼 느껴지도록)
- ⛔ 프로세스를 교체하는 `컨텍스트 스위칭`은 무거운 작업이며, 프로세스는 독립적인 메모리 공간을 가지므로, 프로세스 간 데이터 공유가 까다롭다.

<br>

## 멀티 스레딩

![image](https://github.com/user-attachments/assets/634c0750-17b8-4386-9d4a-154b25939683)

- `하나의 프로세스가 동시에 여러 작업을 실행`한다. (멀티 태스킹처럼 동작하는데, 단위가 스레드인 것!)
> ✅ 스레드의 특징
> - 프로세스는 기본적으로 하나의 스레드를 가진다.
> - 같은 프로세스 간의 컨텍스트 스위칭은 가볍다.
> - 스레드는 자신들이 속한 프로세스의 메모리 영역을 공유한다.

<br>
